Cross-Site Request Forgery (CSRF) Attack

Introduction

Cross-Site Request Forgery (CSRF) is a web security vulnerability that forces a user to perform unwanted actions on a trusted website where they are already authenticated. The attack abuses the trust that a website has in the user’s browser.

What is a CSRF Attack?

A CSRF attack happens when a malicious website sends a request to another website on behalf of a logged-in user. Since the browser automatically sends authentication cookies, the target website believes the request is legitimate.

How CSRF Works

1. A user logs into a trusted website.
2. The website stores authentication data in the browser (cookies).
3. The user visits a malicious website.
4. The malicious site sends a hidden request to the trusted website.
5. The browser includes authentication cookies automatically.
6. The action is executed without the user’s consent.

Example of a CSRF Attack

Consider a website that allows users to change their email using the following link:

https://example.com/change-email?email=new@email.com

If the user is logged in, an attacker can trigger this request from a malicious page, causing the email address to change without permission.

Impact of CSRF Attacks

- Unauthorized money transfers
- Account detail changes
- Data deletion
- Administrative actions

How to Prevent CSRF Attacks

- CSRF tokens
- SameSite cookie attributes
- Checking HTTP Origin or Referer headers
- Re-authentication for sensitive actions

CSRF vs XSS

CSRF forces users to perform actions without their knowledge, while XSS injects malicious scripts into a website. Both are dangerous but work differently.

Conclusion

CSRF is a serious security threat that exploits browser behavior and user trust. Understanding how CSRF works and applying proper protection techniques is essential for building secure web applications.
